# Insect Navigation Simulation (Isaac Sim 4.2)

This repository contains the simulation environment and code for **Insect-inspired Navigation for Central-foraging Robots**.

To ensure full reproducibility of our results, we provide a containerized environment based on Isaac Sim 4.2.0. This allows the simulation to run on Ubuntu Linux without complex local installation.

---

## 1. System Requirements

**Operating System:**
- Ubuntu 20.04 / 22.04

**Hardware:**
- GPU: NVIDIA RTX series (RTX 2070 or higher recommended)
- RAM: 32 GB+ system memory
- Disk Space: ~35 GB free space

**Drivers:**
- NVIDIA Driver version 525.85 or newer

**Software:**
- Docker Engine
- NVIDIA Container Toolkit (for GPU access)

---

## 2. Setup & Installation

### Step 1: Clone the repository and download assets

Clone this repository and navigate to the `visual_simulator_docker` folder.

### Step 2: Download Isaac Sim assets

The 3D assets (tree models, textures, materials) are not included in this repository due to their size (~2.4 GB). Download them separately:

1. Download the `isaac-sim_assets` folder from [Zenodo/SURFdrive link — TODO: insert URL].
2. Extract it into this directory so the folder structure looks like:

```
visual_simulator_docker/
├── isaac-sim_assets/          # Downloaded separately (~2.4 GB)
│   ├── Trees/                 # 50+ tree models (.usd)
│   ├── Bamboo/, Brick_Wall_Red/, Cherry/, ...  # Material/texture assets
│   └── noon_grass_4k.hdr      # HDR environment map
├── Dockerfile
├── run_sim.sh
├── CodeInsectInspired/
└── README.md
```

### Step 3: Build the Docker Image

Open a terminal in this folder (/visual_simulator_docker) and build the environment. This downloads the base simulation engine (Isaac Sim 4.2) and installs all Python dependencies (`tqdm`, `scikit-image`, `onnx`, etc.).

> **Note:** This step may take 15–20 minutes depending on your internet connection.

```bash
docker build -t insect-sim:submission .
```

## 3. Running the experiments

### a. Start the Docker container

Make the script executable:

```bash
chmod +x run_sim.sh
```

Run the simulation:

```bash
./run_sim.sh
```

### b. Running Experiments

Once the container is running, you will see a terminal prompt inside the container (e.g., `root@hostname:/isaac-sim#`).

To run the main experiment:

```bash
cd CodeInsectInspired
../python.sh run_ANN_homing_simulations.py
```


The simulator will start and experiments will begin running.

All config files are set to the correct paths inside the container.

The number of trees is set to 15 by default to accommodate possible hardware limitations. If you have an NVIDIA GPU with more than 4 GB VRAM (e.g., RTX 3050, 2080 Ti), you can change `num_locations = 15` to `num_locations = 40` in each `run_xxx_homing_simulations.py` file (update all three scripts if you change them). Inconsistent values may cause failures.

---

### c. Checking results during and after runtime

When running `run_ANN_homing_simulations.py`, random forest environments (10 envs × 10 runs) are generated by placing trees randomly on a map based on `env3_clean.usd`. Maps are saved to `/CodeInsectInspired/maps`. Maps can be visualized (if drivers permit) in the Isaac Sim popup window—no interaction is required. Only onboard images are captured at different locations; no robot movement/animation is shown.

Onboard images and associated training data are saved to `/CodeInsectInspired/forests`. During execution the terminal displays real-time progress (runs completed and successes per environment). After completion (80 runs per env for 10 envs by default), results are saved to `/CodeInsectInspired/results` and trained models to `/CodeInsectInspired/networks`.

---

### d. Running other experiment scripts


Besides `run_ANN_homing_simulations.py`, you can also run:

- `run_perfect_memory_homing_simulations.py`: Runs homing experiments using perfect memory 
```bash
../python.sh run_perfect_memory_homing_simulations.py
```
- `run_snapshot_homing_simulations.py`: Runs homing experiments using snapshot-based visual navigation
```bash
../python.sh run_snapshot_homing_simulations.py
```

Furthermore, to run mushroombody experiments, in the simple geometric shape environment:

```bash
../python.sh virtual_homing_MB.py
```

This will run homing experiments from multiple start positions at a given radius. Parameters can be modified in `config/config_mb_homing.json`.

## 4. Troubleshooting

**Error:** `docker: Error response from daemon: could not select device driver...`  
**Cause:** NVIDIA Container Toolkit is missing.  
**Fix:** Install the toolkit and restart Docker.

**Error:** `Bus error` or `DataLoader worker (pid x) is killed`  
**Cause:** PyTorch ran out of shared memory.  
**Fix:** Ensure you are using the provided `run_sim.sh` script. This script includes the `--shm-size=8g` flag, which is required for training. Do not run `docker run` manually without this flag.

**Simulation window is black or crashing (Linux)**  
**Cause:** Incompatible X11/Vulkan drivers.  
**Fix:** Try updating your drivers or running in headless mode if available.
